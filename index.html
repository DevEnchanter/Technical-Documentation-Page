Introduction

JavaScript is a versatile and object-oriented scripting language that can run on various platforms. It is known for its lightweight and compact nature. When used within a host environment, such as a web browser, JavaScript interacts with the objects in that environment to enable programmatic control over them.

In addition to its core set of language elements like operators, control structures, and statements, JavaScript comes with a standard library of objects including Array, Date, and Math. Furthermore, JavaScript can be extended by adding extra objects to fulfill specific requirements. Here are a couple of examples:

  • Client-side JavaScript expands the functionality of the core language by providing objects to manipulate and control web browsers and their Document Object Model (DOM). This enables applications to dynamically modify HTML forms, respond to user actions like mouse clicks and form input, and manage page navigation.
  • Server-side JavaScript extends the core language by offering objects that are useful for running JavaScript on servers. With server-side extensions, applications can interact with databases, maintain continuity of information across different invocations of the application, and perform file operations on the server.

JavaScript and Java

While JavaScript and Java share some similarities, they also have fundamental differences. JavaScript bears resemblance to Java in terms of syntax, naming conventions, and basic control-flow constructs. In fact, JavaScript was originally named LiveScript before it was renamed to JavaScript to capitalize on Java's popularity.

However, JavaScript diverges from Java in several key aspects. Unlike Java, JavaScript does not have static typing and strong type checking. Instead, it employs a runtime system with a limited number of data types for numeric, Boolean, and string values. JavaScript adopts a prototype-based object model rather than the more traditional class-based model found in Java. This prototype-based approach allows for dynamic inheritance, where objects can vary in their inherited properties.

In JavaScript, functions hold a unique position. They are not bound by strict declarative requirements and can be used as properties of objects, functioning as loosely typed methods.

In contrast to the strict structure and requirements of Java, JavaScript offers a more flexible and free-form language experience. There is no obligation to declare all variables, classes, or methods. Concerns regarding access modifiers like public, private, or protected are not present in JavaScript, and interfaces are not implemented. Furthermore, explicit typing is not required for variables, parameters, or function return types.

Overall, JavaScript provides a more relaxed and dynamic programming environment compared to the stricter and more structured nature of Java.

Hello world

To begin writing JavaScript code, follow these steps:

  • Open the Scratchpad, which is a built-in tool in most web browsers.
  • Write the following code to display a "Hello world" message:

function greetMe(yourName) {
  alert("Hello " + yourName);
}
greetMe("World");

Select the code within the Scratchpad.

Press Ctrl+R (or the equivalent key combination) to run the code and see it in action within your web browser.

By executing the code, a dialog box will appear displaying the message "Hello World." This serves as a simple introduction to JavaScript and demonstrates how to create and call a function. Feel free to modify the code and explore further possibilities with JavaScript.

Variables

In JavaScript, variables are used to represent values within your application. These variables are given names, known as identifiers, which must follow certain rules.

An identifier in JavaScript should begin with a letter, underscore (_), or dollar sign ($). Subsequent characters can also be digits (0-9). It's important to note that JavaScript is case-sensitive, distinguishing between uppercase letters ("A" to "Z") and lowercase letters ("a" to "z").

In identifiers, you can utilize ISO 8859-1 or Unicode letters, including special characters like "å" and "ü". Additionally, Unicode escape sequences can be used as characters in identifiers. Here are some examples of valid identifier names: Number_hits, temp99, and _name.
